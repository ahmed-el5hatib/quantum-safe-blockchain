# QSB v0.3.0 — Cryptography Foundation

**Release Date**: 2026-07-13  
**Release Name**: Cryptography Foundation

## Overview

QSB v0.3.0 delivers the first production-ready component of the Quantum Safe Blockchain: a modular, crypto-agile cryptography framework. This release establishes the cryptographic backbone upon which all future blockchain functionality will be built.

## Major Achievements

- **Provider Architecture**: Complete separation of cryptographic interfaces from implementations. The blockchain never knows that Ed25519 or SHA-256 exist.
- **Crypto Agility**: Future post-quantum algorithms (ML-DSA, ML-KEM, Falcon, SPHINCS+) can be added as new providers without modifying any blockchain logic.
- **Production Quality**: Comprehensive testing, benchmarking, documentation, and security hardening.

## Architecture Status

The cryptography layer is fully implemented and validated:

```text
cryptography/
├── core/
│   ├── errors/       # CryptoError, CryptoResult
│   ├── traits/       # HashFunction, SignatureAlgorithm, PublicKey, PrivateKey, Signature, RandomGenerator, Encoder, Decoder
│   ├── key/          # HashDigest, KeyPair
│   ├── encoding/     # HexEncoder, HexDecoder, Base64Encoder, Base64Decoder
│   ├── random/       # StdRngGenerator (OS-level RNG)
│   ├── hash/         # HashFunction trait re-export
│   └── signature/    # SignatureAlgorithm trait re-export
└── providers/
    ├── sha256.rs     # SHA-256 hash provider
    └── ed25519.rs    # Ed25519 signature provider
```

## Implemented Features

### Cryptography Providers

#### Hash Providers
- **SHA-256**: Deterministic hashing with `hash()`, `hash_bytes()`, `hash_stream()`, `digest_size()`, and `algorithm_name()`

#### Signature Providers
- **Ed25519**: Key generation, signing, verification, serialization/deserialization, byte export/import

### Key Types
- Strongly typed wrappers: `PublicKey`, `PrivateKey`, `Signature`, `HashDigest`, `KeyPair`
- `Display`, `Debug`, `Clone`, `Eq`, `Serde` support where appropriate
- Private keys implement memory zeroization on drop

### Encoding
- Hexadecimal encoding/decoding
- Base64 encoding/decoding

### Randomness
- OS-level cryptographically secure RNG via `rand_core::OsRng`

### Error Handling
- Comprehensive `CryptoError` enum with `thiserror`
- All operations return `CryptoResult<T>`

## Testing

- **25 unit tests** covering:
  - Hash determinism and uniqueness
  - Key generation and roundtrip serialization
  - Signature verification, wrong key, wrong message
  - Empty and large message edge cases
  - Repeated signing determinism
  - Invalid input handling
- **9 doctests** validating all public examples
- All tests pass on stable Rust 1.75.0

## Benchmarks

- SHA-256 hashing (small and large inputs)
- Ed25519 key generation, signing, and verification
- Run with: `cargo bench -p cryptography`

## Known Limitations

- Full workspace build is blocked on Windows due to missing `libclang` (required by `bindgen` for native dependencies such as RocksDB). Pure-Rust crates (`cryptography`, `blockchain-core`) build and test successfully.
- Only classical cryptography is implemented. Post-quantum algorithms are planned for future milestones.
- No blockchain logic, networking, consensus, or wallet functionality is implemented yet.

## What's Next

The next milestone will focus on implementing the Blockchain Core layer:

- Block and transaction types
- Chain validation rules
- State management
- Merkle tree abstractions

Future milestones will add networking, consensus, wallet, RPC, and explorer functionality.
